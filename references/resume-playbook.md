# 改简历 Playbook

> 一句话：**先锁定目标岗位要什么，再把候选人已有的真实经历，翻译成那套语言。** 不动真实经历，只改讲它的角度。

## 为什么多数简历改不出效果

- 堆关键词：塞满"AI/增长/数据驱动"，但每条经历还是流水账，一看就空。
- 照时间平铺：不区分哪条对岗位重要，让面试官自己找匹配点——他不会找。
- 为对口去编：编一段没做过的，面试一追问就崩。

正确做法 = **目标岗位反向定位 + 能力迁移翻译**。

## 五步法

### 1. 目标定位
拿 Stage 1 岗位调研里的 **3-5 个核心能力关键词** 当靶心。整份简历反复命中它们。

### 2. 盘点真实经历
把所有经历摊开（实习、项目、研究、社团、个人作品都算）。每段问三件事：
- 真实做了什么？（不是简历上写的，是真做的）
- 哪些细节、过程、数据、产物能在面试里讲清？只是用户口头说过但没有细节支撑的，标成"待确认"。
- 哪部分能对应到某个靶心？

建立"真实经历 → 目标能力 → 证据置信度"的映射。没映射的经历，弱化或砍掉；没证据的 claim，降级、模糊化或删掉。

### 3. 写求职意向头（标志性动作）
简历最顶部加一段目标导向定位句：

```
求职意向：[目标岗位] | [时间/身份] | [学历背景]，
具备[3-5 个能力关键词]，曾[一句话最强/最对口经历]，
希望在[目标公司/方向]……
```

作用：面试官 5 秒内看到"这人冲我们岗位来的"，而不是"一个 X 专业的来投 Y，奇怪"。

### 4. 能力迁移翻译（最核心）
逐条改写，三段式：**真事 → 用目标岗位语言重述 → 必要时自然点明可迁移性。**

迁移表达是跨行工具，不是每条经历都要贴的模板。用法要克制：
- 最对口的经历不需要解释"为什么相关"，让结果和关键词自己说话。
- 明显跨行、但确实有可迁移能力的经历，可以在 1 条 bullet 末尾自然点明。
- 一份简历里通常 1-2 处就够；连续每段都写"可直接迁移至..."会显得公式化、偏长。
- 绝不要在最终简历里出现"迁移句:""迁移说明:"这种标签词。

例（教育学经历 → 金融岗）：
> 原："追踪用户订单量、转化率等核心指标，为营销提供数据支持"
> 改后："……追踪订单量、转化率等核心指标，整理转化漏斗，把用户分层、数据追踪与服务意识转化为客户运营可复用的方法。"

翻译的边界（诚实底线）：
- ✅ 可以：换角度、换术语、点明可迁移性
- ❌ 不可以：把没做的事写成做了、迁移一个根本不存在的能力、把"迁移句:"当标签印进 bullet

### 5. 重排序 + 量化 + 砍无关
- **重排序**：最对口的经历提到最前，单开"核心经历/核心项目"；次要的放后面。
- **量化**：尽量带数字。真实的用真数；讲不清来源的，模糊化（"双位数提升""测算方式记不准了"），**不编造**。
- **砍无关**：与目标岗位无关的内容删掉（如兴趣爱好里的无关项）。

## 常见结构模板

```
姓名 + 一行联系方式
求职意向：[定位头]
教育背景
核心经历 / 核心项目      ← 最对口的，放最前；跨行明显时少量点明可迁移性
实习经历                ← 次对口的
其他项目 / 研究经历
技能                    ← 按目标岗位重组分类（如"市场与增长""金融服务"）
```

## 输出 docx
内容定稿后用 `scripts/generate_resume.py` 渲染成统一模板。把改好的内容整理成 JSON（结构见 `scripts/sample_resume.json`）后运行脚本。

## 自查清单
- [ ] 顶部有目标导向的求职意向头
- [ ] 3-5 个能力关键词在简历里反复命中
- [ ] 关键跨行经历已自然点明可迁移性，但没有每段机械挂一句
- [ ] 最终 bullet 里没有"迁移句:""迁移说明:"等标签词
- [ ] 最对口的经历排在最前
- [ ] 讲不清来源的数字/经历已模糊化或删除，没有编造
- [ ] 学历/在职/职位名等背调硬信息一字未改
