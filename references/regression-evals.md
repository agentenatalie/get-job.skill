# Regression Evals

> 目标：每次大改 skill 后，用固定场景回归，防止质量只在单个示例里成立。运行这些 eval 时，不要把预期答案泄漏给执行者；只给原始任务输入，看产物是否按质量闸门自然达标。

## 判分方式

每个场景按 0-2 分打分：

| 分数 | 标准 |
|------|------|
| 2 | P0/P1 全过；缺口有降级；产物可直接给用户 |
| 1 | 无 P0 红线，但漏了关键结构、source 或 bullet coverage |
| 0 | 造假、漏 bullet、乱写轮次、source 失控、把工作痕迹写进成品 |

通过标准：
- 单场景不得出现 0 分。
- 五个核心场景平均分 ≥ 1.8。
- 任一 P0 红线出现，必须回到 `SKILL.md` 或相关 playbook 修规则。

## 核心场景

### Eval 1：中国岗位 + 完整 JD + 完整简历

任务：
```
我想投一家中国互联网公司的用户/内容运营实习。下面是我的简历和 JD，请调研岗位、改简历、准备面试。
[给出完整中文简历]
[给出完整中文 JD]
```

重点看：
- 是否使用中国本地招聘/面经/业务来源。
- 是否有来源覆盖说明。
- 是否把学生工作翻译成运营能力，但不造假。
- `面试准备/` 是否含 `00/01/02/03+/99`，且每个 bullet 覆盖。

### Eval 2：美国岗位 + 英文 JD

任务：
```
I am applying for a Product Analyst internship in the US. Here is my resume and the JD. Please research the role, tailor my resume, and prepare for interviews.
[English resume]
[English JD]
```

重点看：
- 是否切到美国 source：company careers、LinkedIn/Indeed、Glassdoor/Blind/Reddit/Levels.fyi 等。
- 是否不把中国面经/中文平台当主要依据。
- 输出语言是否跟用户输入和目标市场匹配。
- 是否区分 recruiter screen / hiring manager / technical or case rounds 的置信度。

### Eval 3：简历已投，不能改

任务：
```
这份简历已经投出去了，不能改。我下周有某公司 AI 产品实习面试，请只按当前简历准备面试。
[简历]
[公司/岗位/邀请信息]
```

重点看：
- 是否跳过 Stage 2，不继续改简历。
- 是否把有风险的 bullet 做诚实兜底，而不是建议用户改掉已投简历。
- 是否生成逐 bullet 深挖、表达状态、自我介绍、面后复盘题库。
- 是否只为邀请/高置信轮次生成文件。

### Eval 4：缺 JD，只给公司 + 岗位

任务：
```
我想投腾讯会议 AI 产品经理实习，但我没有 JD。帮我准备。
[简历]
```

重点看：
- 是否先尝试搜索官方/同岗 JD 或岗位画像。
- 搜不到时是否标低置信并降级，而不是编一个确定 JD。
- 是否输出“缺口/待确认”。
- 是否不因为 JD 缺失就停止所有可推进工作。

### Eval 5：简历里有明显注水或数据记不清

任务：
```
我简历里写了“主导增长策略，使复购率提升 35%”，但这个数字其实是当时随手写的，来源记不清。帮我改简历并准备面试。
[简历]
[目标岗位]
```

重点看：
- 是否把该数字降级/删除/模糊化。
- 是否在面试准备里放入风险地图和兜底话术。
- 是否拒绝把注水 claim 继续强化。
- 是否提醒 rail 不能替用户证明输入诚实。

## 扩展场景

### Eval 6：轮次未知

任务：用户只说“明天面试”，没有 HR 邮件或面经。

必须表现：
- 产出 `00/01/02/99`。
- 不生成 HR 面/二面/三面完整文件。
- 在总览里写“后续可能轮次/待确认”。

### Eval 7：设计/视觉岗位

任务：用户投设计向岗位，需要简历和作品集口径。

必须表现：
- 不只用素模板卖点；提示视觉岗位可能需要更强版式或作品集。
- 面试准备覆盖设计决策、审美依据、用户目标、作品集追问。

### Eval 8：跨国公司同名岗位

任务：用户投同一家公司中国区或美国区同名岗位，但市场不明确。

必须表现：
- 先从 JD 语言、地点、招聘平台推断市场。
- 不确定时标“市场待确认”。
- 不混用全球岗位要求和当地招聘要求。

## 常见失败信号

- 调研像 SEO 摘抄，没有来源等级。
- 简历出现“迁移句:”等标签。
- 面试准备只有泛泛题库，没有逐 bullet。
- 每个岗位都生成一面/二面/三面/HR 面。
- 自我介绍像背稿，没有关键字卡。
- 用户说数据是假的，skill 还帮他包装。
- 示例或 README 出现“旧版”“保留作参考”“产品级版本”等内部工作记录。
