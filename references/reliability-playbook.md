# 可靠性 Playbook

> 目标：减少"输入不完整就停住"和"边界情况全靠人工确认"。先自动体检、补齐、降级，再决定是否必须问用户。

## 一、输入体检

先把用户输入整理成 `Input Health` 表：

| 字段 | 状态 | 自动处理 |
|------|------|----------|
| 目标公司 / 岗位 | ✅ / ⚠️ / 🛑 | 缺公司但有岗位：做通用岗位准备；缺岗位：必须问 |
| 目标市场 / 语言 | ✅ / ⚠️ / 🛑 | 从 JD 语言、公司地点、招聘平台推断；不确定标低置信 |
| JD / 招聘信息 | ✅ / ⚠️ / 🛑 | 缺 JD 时用 WebSearch 找官方招聘页/同岗 JD |
| 简历正文 | ✅ / ⚠️ / 🛑 | 文件解析失败则请用户粘贴；只给片段时只处理片段并列缺口 |
| 投递状态 | ✅ / ⚠️ / 🛑 | 未知时默认"未确认"，改简历前必须提醒用户确认是否已投 |
| 面试轮次 / 时间 | ✅ / ⚠️ / 🛑 | 从邀请、面经、公司流程推断；推不出不生成具体轮次文件 |

状态定义：
- **✅ Complete**：足够直接产出。
- **⚠️ Workable**：能继续，但必须在产物里标"假设/缺口/低置信"。
- **🛑 Blocked**：继续会编造或误导，只问一个最关键问题。

## 二、自动恢复策略

### 缺 JD
1. 有公司+岗位：优先搜公司官网/招聘页，其次同岗位 JD，再次同类岗位。
2. 找不到官方 JD：用通用岗位画像继续，但在 `岗位调研.md` 标"JD 缺失，结论低置信"。
3. 不允许把通用岗位画像写成该公司的真实要求。

### 缺目标市场
1. 从 JD 语言、地点、货币、招聘平台、公司实体推断。
2. 同时存在多个市场时，分市场列假设，不混成一个结论。
3. 无法判断时，问用户"目标国家/地区是哪一个"，不要继续深搜错市场。

### 简历不完整
1. 只对已提供内容改写和面试准备。
2. 新增 `素材缺口清单`：缺教育、缺项目结果、缺时间线、缺指标口径、缺产物链接等。
3. 不因为缺结果就编数字；不因为缺角色就写"主导/负责"。

### 轮次不明
1. 先从用户邀请、面经、公司流程找证据。
2. 只为 ✅ 已确认或 ⚠️ 高置信轮次生成单独文件。
3. 无轮次证据时，只产出 `00-总览.md` 与 `01-简历bullet逐条深挖.md`，并在总览列"后续可能轮次/待确认"。

### 搜索源不足或 SEO 污染
1. 来源降级：官方/JD > 一手面经 > 可靠二手 > SEO 软文。
2. 只有 D 级来源时，不得给强结论，只能写"线索"。
3. 产物必须包含"来源覆盖说明"和"缺口"。

### 用户 claim 没证据
1. 标为 ⚠️ 低置信或 🚨 风险。
2. 简历未投：回到 Stage 2 降级/删除。
3. 简历已投：Stage 3 准备诚实兜底话术。

## 三、产出降级模式

| 模式 | 适用情况 | 产出 |
|------|----------|------|
| Full | JD、简历、目标市场、轮次信息充足 | 完整三阶段产物 |
| JD-lite | 缺官方 JD，但公司+岗位明确 | 调研低置信标注 + 简历/面试正常推进 |
| Resume-lite | 简历只有片段 | 只处理片段 + 素材缺口清单 |
| Interview-lite | 简历已投但轮次未知 | `00-总览.md` + `01-简历bullet逐条深挖.md` |
| Blocked | 缺岗位或缺简历正文 | 停止生成，问最少必要问题 |

## 四、可靠性完成标准

- [ ] 每次任务开头都能说明输入状态：Complete / Workable / Blocked。
- [ ] Workable 状态下继续产出，但所有假设、缺口、低置信结论都明确标注。
- [ ] 不因缺 JD、缺轮次、缺指标口径而编造内容。
- [ ] 至少尝试一次自动补齐：抽取、搜索、推断或降级。
- [ ] 只有真正 Blocked 时才向用户提问。
- [ ] 最终产物里有"缺口/待确认"区域，用户知道后续要补什么。
