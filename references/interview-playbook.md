# 面试准备 Playbook

> 简历改好只让对方愿意见你，面试还得逐题备好、逐 bullet 深挖、且被追问不崩。本 playbook 四步：建题库 → 简历 bullet 逐条深挖 → 证据化审计拆雷 → 话术。产出 `面试准备.md`。

---

## 一、建题库

抓**对口的真实面经**，去重后按出现频率分级。

**先按目标市场选源**，不要默认只搜中文或只搜英文：

| 市场 | 面经 / 口碑源 | JD / 岗位源 | 业务背景源 |
|------|---------------|-------------|------------|
| 中国大陆 | 用户提供面经、牛客、脉脉、小红书、看准、offershow | 公司官网、校招/社招官网、BOSS 直聘、拉勾、猎聘、前程无忧、智联招聘 | 公司官网、公众号/新闻稿、36氪/晚点/行业报告 |
| 美国 | 用户提供面经、Glassdoor、Blind、Reddit、Levels.fyi、校友材料 | Company careers、Greenhouse、Lever、LinkedIn Jobs、Indeed | 公司官网、产品文档、SEC/投资者材料、TechCrunch/主流媒体 |
| 其他市场 | 用户提供材料、当地论坛/Glassdoor/校友面经 | 公司官网、LinkedIn、当地头部 job board | 公司官网、当地媒体、行业报告 |

**来源优先级**：用户自己提供的面经 > 目标市场的一手面经/口碑源 > 公司官网/发布会 > WebSearch 兜底。环境里装了采集类 skill（如 `tikhub-xhs`）就优先调用。

**筛选**：对口度（同部门同岗位最高）+ 时效（近 6 个月）+ 热度，挑 5-10 篇深读。**凉经必看**——挂掉的人会写清自己栽在哪。

**分级**：
- 🔴 必考（≥3 篇出现）：自我介绍、简历最深的项目、为什么转岗/为什么我们公司
- 🟡 高频（2 篇）：领域基础题、协同软实力场景题
- 🟢 中频（1 篇）：开放设计题、行业判断题
- 🔁 反问（必问）：准备 3 个，现场用 2-3 个

附产出：**凉经雷区清单**（挂掉的人都栽在哪，常见——项目讲成流水账缺量化、被追问数据就崩、硬装懂被问穿）。

## 二、简历 bullet 逐条深挖（护城河）

这是 Stage 3 的核心交付。面试官最常见的深挖入口不是泛泛题库，而是简历上的每一个 bullet。必须把最终简历拆成 bullet inventory，逐条编号覆盖。

### Bullet inventory
按简历顺序编号，例如：

```
P1-B1 核心项目 / AI 记账小工具 / 痛点与方案
P1-B2 核心项目 / AI 记账小工具 / 产品决策
E1-B1 实习经历 / 数据运营 / 数据跟踪与复盘
```

每个 bullet 至少覆盖：

| 字段 | 要求 |
|------|------|
| Bullet 原文 | 直接贴最终简历里的这条，避免漏项 |
| 面试官可能怎么问 | 2-4 个高概率追问，按岗位和面经定制 |
| 用户必须讲清的事实 | 背景、目标、角色、动作、工具/方法、时间线、协作者、结果 |
| 证据状态 | ✅能讲清 / ⚠️部分记不清 / 🚨不能硬讲 |
| 安全答法 | 30-60 秒口语化回答，不要写论文 |
| 二三层追问 | 至少 1 个"如果继续深挖"的问题和兜底 |
| 处理策略 | 放大 / 正常讲 / 降级 / 转场 / 简历未投则回改 |

### 高价值项目深挖维度
对王牌项目和高风险 bullet，要额外准备这些细节：
- **场景**：谁遇到了什么问题，为什么值得做？
- **目标**：当时怎么定义成功，指标或验收标准是什么？
- **角色**：你具体 own 哪部分，别人做了什么？
- **方案取舍**：至少 2 个关键 trade-off，为什么选 A 不选 B？
- **执行过程**：时间线、工具、方法、协作对象、遇到的阻力。
- **结果证据**：数字怎么算、样本多大、基线是什么、有没有截图/链接/产物。
- **复盘**：如果重做，会改什么？这段经历如何对应目标岗位？

### Definition of Done
- [ ] 最终简历每个 bullet 都出现在 bullet inventory 里。
- [ ] 每个 bullet 至少有 2 个可能追问、1 个安全答法、1 个兜底。
- [ ] 每个数字都有口径：怎么算、样本/时间范围、基线。
- [ ] 每个"主导/负责/独立/提升/优化"都有角色边界和证据。
- [ ] 用户讲不清的 bullet 已标风险；简历未投就回到 Stage 2 改写/删除，已投就准备诚实降级话术。

## 三、证据化审计 + 风险地图（防崩核心）

面试官会顺着简历追问两三层。这一步在面试**之前**把雷拆掉。

### 逐条 claim 拆雷
基于 bullet inventory，把简历上每个数字、动作、职责边界和成果列出来，逐个问用户：
> "这个被追问『怎么算的？样本多大？对比基线？你具体做了什么？』，你能讲清吗？"

分三类：
- ✅ **真实能讲清** → 可放大的弹药，用 STAR-R 讲到极致
- ⚠️ **部分真/记不清** → 数字模糊化（"双位数提升""测算方式记不准了"），诚实承认记忆边界
- 🚨 **假/编/心虚** → 短答 + 模糊 + 主动转走，不主动加细节（加得越多崩得越快）

### 风险地图（Stage 3 核心产出）
| 项目 | 状态 | 处理策略 | 时间预算 |
|------|------|---------|---------|
| 王牌项目 | ✅ 真做有数据 | 重头戏，强调产品/业务决策 | 40-50% |
| 某实习 | 🚨 记不清/数字假 | 缩短+模糊化+主动转走 | <2 分钟 |
| 某经历 | ⚠️ 真但不对口 | 短答+主动承认相关度低 | <2 分钟 |

### 红线
- 不编造不存在的经历或数据
- 不动学历/在职/职位名等背调硬信息
- 模糊化都是承认记不清，不是新编一个谎

## 四、话术生成

### 自我介绍钩子（30 秒）
- 每个字都真，被任何追问都不崩
- 开头放最强钩子（王牌项目 + 一个亮眼真实数字）
- 针对岗位方向定制（技术与产品方向突出技术理解 + ToB 思维，别讲 C 端故事感）

### 项目 STAR-R（每个王牌项目 60-90 秒）
**痛点（用户/场景/问题）→ 决策（2-3 个关键取舍 + 为什么）→ 执行（诚实说自己的角色）→ 结果（真实量化）→ 反思（产品认知）。**
决策写法区分高手和流水账：不说"我做了 X 功能"，说"我在 X 和 Y 之间选了 X，因为 Z"。

### Bullet 追问兜底表（防崩）
把第二三层问题提前备好，统一收敛到两条安全线：
| 追问类型 | 安全应答 |
|---------|---------|
| 技术实现细节 | "这部分我做的是产品/架构决策，原理大概是 X，但代码是 AI 辅助实现的，不是我从头敲的" |
| 数字怎么算 | "比起这个数字，我更想讲当时怎么做、为什么、业务上发生了什么改变" |
| 心虚经历被深挖 | 短答一句 + "这段时间短收获有限，我更想聊我后面做的王牌项目" |

> "AI 辅助"在今天是加分项，装工程师被追问三层一定崩。

### 软实力场景题（ToB/产品岗高频）
"需求 push 不动研发""研发说做不了""需求频繁变更"——核心：理解对方立场 + 用数据对齐 + 务实取舍，别答得太理想化。

### 反问（挑 2-3 个）
"这个岗位/组未来半年最重要的目标？" / "做得好的人最关键的特质？" / 针对具体产品："你们最近发的 X，背后的产品考量是？"
避免：一面问薪资/转正率、问能查到的基础信息。

## 五、面试准备文档结构

把以上组装成面试前 30 分钟能快速过的 `面试准备.md`。原则：**能背、能扫、认知负荷低**，多表格少长段落。

```
1. 只能记 3 条就记这个
2. 自我介绍（背诵版）
3. 简历 bullet 逐条深挖表（每条 bullet 都覆盖）
4. 风险地图 + 各项目话术和兜底表
5. 分级题库 + 答题要点
6. 领域速记（概念 + 竞品 + 最新动态，来自 Stage 1 调研）
7. 软实力场景题模板
8. 反问清单
9. 开场前 5 分钟 checklist
```

需要 PDF 用 pandoc 或 `pdf` skill 转；默认交付 Markdown。
