# 面试准备 Playbook

> 简历改好只让对方愿意见你，面试还得按真实轮次备好、逐 bullet 讲明白、且被追问不崩。本 playbook 五步：建题库与轮次发现 → 简历 bullet 逐条深挖 → 证据化审计拆雷 → 按实际轮次出文件 → 话术。产出 `面试准备/` 文件夹。

---

## 一、建题库 + 发现轮次地图

抓**对口的真实面经**，去重后按出现频率分级；同时发现这个岗位实际有几轮、每轮谁面、每轮看什么。不要默认所有岗位都有一面/二面/三面/HR 面，也不要因为模板里有某轮就硬生成。

**先按目标市场选源**，不要默认只搜中文或只搜英文：

| 市场 | 面经 / 口碑源 | JD / 岗位源 | 业务背景源 |
|------|---------------|-------------|------------|
| 中国大陆 | 用户提供面经、牛客、脉脉、小红书、看准、offershow | 公司官网、校招/社招官网、BOSS 直聘、拉勾、猎聘、前程无忧、智联招聘 | 公司官网、公众号/新闻稿、36氪/晚点/行业报告 |
| 美国 | 用户提供面经、Glassdoor、Blind、Reddit、Levels.fyi、校友材料 | Company careers、Greenhouse、Lever、LinkedIn Jobs、Indeed | 公司官网、产品文档、SEC/投资者材料、TechCrunch/主流媒体 |
| 其他市场 | 用户提供材料、当地论坛/Glassdoor/校友面经 | 公司官网、LinkedIn、当地头部 job board | 公司官网、当地媒体、行业报告 |

**来源优先级**：用户自己提供的面经 > 目标市场的一手面经/口碑源 > 公司官网/发布会 > WebSearch 兜底。环境里装了采集类 skill（如 `tikhub-xhs`）就优先调用。

**筛选**：对口度（同部门同岗位最高）+ 时效（近 6 个月）+ 热度，挑 5-10 篇深读。**凉经必看**——挂掉的人会写清自己栽在哪。

### 轮次发现规则
优先级：用户收到的邀请/邮件/HR 信息 > 用户提供的面经 > 同公司同岗位近 6 个月面经 > 公司招聘流程说明 > 同类岗位经验推断。

每个轮次都要标置信度：
- ✅ **已确认**：用户邀请或明确面经写到。
- ⚠️ **高置信推断**：多个近期面经都出现，或公司流程明确。
- ❓ **待确认**：只有少量线索或同类岗位推断。待确认轮次可在总览里提醒，但不单独生成完整话术文件，除非用户要求预备。

### 轮次地图输出

| 轮次 | 依据 | 置信度 | 可能面试官 | 主要看什么 | 是否生成文件 |
|------|------|--------|------------|------------|--------------|
| 例：业务一面 | 用户邀请写"业务一面" | ✅ 已确认 | 直属组员 / mentor | 简历真实性、项目细节 | 是 |
| 例：产品负责人面 | 3 篇近期面经均出现 | ⚠️ 高置信推断 | hiring manager | 业务理解、产品判断 | 是 |
| 例：HR 面 | 只有同类岗位常见 | ❓ 待确认 | HR | 动机、稳定性 | 只在总览提醒，不单独生成 |

### 常见轮次类型参考
这些只是命名参考，不是必须生成：
- **初面 / 业务一面 / 技术一面**：通常深挖简历真实性、项目细节、岗位基础能力。
- **专业面 / 主管面 / 产品负责人面**：通常看业务理解、产品判断、竞品意识、方法论。
- **交叉面 / 终面 / 高阶面**：通常看综合判断、潜力、取舍、组织协作。
- **HR 面 / recruiter screen**：通常看动机、稳定性、薪资/到岗、风险点；只有被邀请或调研高置信时才单独成文。

**分级**：
- 🔴 必考（≥3 篇出现）：自我介绍、简历最深的项目、为什么转岗/为什么我们公司
- 🟡 高频（2 篇）：领域基础题、协同软实力场景题
- 🟢 中频（1 篇）：开放设计题、行业判断题
- 🔁 反问（必问）：准备 3 个，现场用 2-3 个

附产出：**凉经雷区清单**（挂掉的人都栽在哪，常见——项目讲成流水账缺量化、被追问数据就崩、硬装懂被问穿）。

## 二、简历 bullet 逐条深挖（护城河）

这是 Stage 3 的核心交付。面试官最常见的深挖入口不是泛泛题库，而是简历上的每一个 bullet。必须把最终简历拆成 bullet inventory，逐条编号覆盖，并为每条写出候选人能直接说出口的讲法。

### Bullet inventory
按简历顺序编号，例如：

```
P1-B1 核心项目 / AI 记账小工具 / 痛点与方案
P1-B2 核心项目 / AI 记账小工具 / 产品决策
E1-B1 实习经历 / 数据运营 / 数据跟踪与复盘
```

每个 bullet 至少覆盖：

| 字段 | 要求 |
|------|------|
| Bullet 原文 | 直接贴最终简历里的这条，避免漏项 |
| 对应岗位问题 | 这条证明目标岗位需要的哪种能力，能解决公司什么问题 |
| 这条该怎么讲 | 30 秒口语版，按"背景 → 动作 → 结果/反思"讲清楚 |
| 60-90 秒展开版 | 对王牌/高风险 bullet，补充决策、过程、取舍和证据 |
| 面试官可能怎么问 | 2-4 个高概率追问，按岗位、轮次和面经定制 |
| 用户必须讲清的事实 | 背景、目标、角色、动作、工具/方法、时间线、协作者、结果 |
| 证据状态 | ✅能讲清 / ⚠️部分记不清 / 🚨不能硬讲 |
| 数据/证据口径 | 数字怎么算、样本哪来、基线是什么、产物/截图/链接在哪里 |
| 二三层追问 | 至少 1 个"如果继续深挖"的问题和兜底 |
| 不能说什么 | 容易露馅或过度包装的话，明确禁止 |
| 处理策略 | 放大 / 正常讲 / 降级 / 转场 / 简历未投则回改 |

### 高价值项目深挖维度
对王牌项目和高风险 bullet，要额外准备这些细节：
- **场景**：谁遇到了什么问题，为什么值得做？
- **目标**：当时怎么定义成功，指标或验收标准是什么？
- **角色**：你具体 own 哪部分，别人做了什么？
- **方案取舍**：至少 2 个关键 trade-off，为什么选 A 不选 B？
- **执行过程**：时间线、工具、方法、协作对象、遇到的阻力。
- **结果证据**：数字怎么算、样本多大、基线是什么、有没有截图/链接/产物。
- **复盘**：如果重做，会改什么？这段经历如何对应目标岗位？

### 常见 bullet 深挖模板
- **问卷 / 调研**：为什么这么设计？样本从哪来？样本偏差怎么处理？结论怎么验证？有没有影响后续决策？
- **数据分析 / 指标提升**：指标口径是什么？基线是什么？提升怎么归因？有没有对照组？你具体做了哪一步？
- **产品 / AI 项目**：用户是谁？痛点怎么发现？为什么这样设计？竞品怎么做？AI 的边界和风险是什么？
- **运营 / 活动**：目标是什么？漏斗哪一步掉了？你怎么定位问题？复盘后改了什么？
- **咨询 / 心理 / 敏感记录**：如何匿名化？隐私边界是什么？数据安全怎么处理？哪些不能被产品化？

### Definition of Done
- [ ] 最终简历每个 bullet 都出现在 bullet inventory 里。
- [ ] 每个 bullet 至少有 1 个 30 秒讲法、2 个可能追问、1 个兜底。
- [ ] 每个数字都有口径：怎么算、样本/时间范围、基线。
- [ ] 每个"主导/负责/独立/提升/优化"都有角色边界和证据。
- [ ] 每个高价值 bullet 都有 60-90 秒展开版。
- [ ] 用户讲不清的 bullet 已标风险；简历未投就回到 Stage 2 改写/删除，已投就准备诚实降级话术。

## 三、证据化审计 + 风险地图（防崩核心）

面试官会顺着简历追问两三层。这一步在面试**之前**把雷拆掉。

### 逐条 claim 拆雷
基于 bullet inventory，把简历上每个数字、动作、职责边界和成果列出来，逐个问用户：
> "这个被追问『怎么算的？样本多大？对比基线？你具体做了什么？』，你能讲清吗？"

分三类：
- ✅ **真实能讲清** → 可放大的弹药，用 STAR-R 讲到极致
- ⚠️ **部分真/记不清** → 数字模糊化（"双位数提升""测算方式记不准了"），诚实承认记忆边界
- 🚨 **假/编/心虚** → 短答 + 模糊 + 主动转走，不主动加细节（加得越多崩得越快）

### 风险地图（Stage 3 核心产出）
| 项目 | 状态 | 处理策略 | 时间预算 |
|------|------|---------|---------|
| 王牌项目 | ✅ 真做有数据 | 重头戏，强调产品/业务决策 | 40-50% |
| 某实习 | 🚨 记不清/数字假 | 缩短+模糊化+主动转走 | <2 分钟 |
| 某经历 | ⚠️ 真但不对口 | 短答+主动承认相关度低 | <2 分钟 |

### 红线
- 不编造不存在的经历或数据
- 不动学历/在职/职位名等背调硬信息
- 模糊化都是承认记不清，不是新编一个谎

## 四、按轮次出文件

默认产出一个 `面试准备/` 文件夹，而不是把所有东西塞进单个长文档。每个文件都要能在对应轮次前 30 分钟扫完。文件数量由轮次发现结果决定，不写死。

```
面试准备/
├── 00-总览.md
├── 01-简历bullet逐条深挖.md
├── 02-{已确认或高置信轮次名}.md
└── ...
```

### `00-总览.md`
- 只能记 3 条就记这个
- 轮次地图：依据、置信度、面试官、考察重点、是否生成文件
- 风险地图：哪段能放大、哪句危险、哪些数字不能硬报
- Bullet coverage checklist：所有 bullet 是否已覆盖

### `01-简历bullet逐条深挖.md`
这是必产文件，和轮次无关。只要用户要准备面试，就必须把最终简历所有 bullet 放进这个文件。
- Bullet inventory：每条 bullet 的 ID、原文、对应岗位能力。
- 对应讲解：每条 30 秒口语讲法；高价值 bullet 另写 60-90 秒展开版。
- 证据口径：数字怎么算、样本哪来、基线是什么、产物在哪里。
- 深挖问题：每条至少 2 个追问、1 个二三层追问、1 个兜底。
- 禁区：这条不能怎么讲、哪些词会显得过度包装。

### 轮次文件怎么写
根据实际轮次命名，例如 `02-业务一面-简历深挖.md`、`03-产品负责人面-业务判断.md`、`02-recruiter-screen.md`。每个轮次文件包含：
- 本轮 30-60 秒自我介绍：按该轮关注点定制。
- 本轮最可能问的问题：来自 JD、产品线、竞品、面经。
- 本轮要用的 bullet 讲法：引用 `01-简历bullet逐条深挖.md` 的 bullet ID，不复制全部内容；只突出本轮会深挖的部分。
- 风险句和兜底：本轮最容易露馅的地方。
- 本轮反问。

### 简历深挖轮常见内容
- 30 秒自我介绍：偏简历真实性和岗位匹配
- 每个 bullet 的 30 秒讲法、二三层追问、证据口径、不能说什么
- 王牌项目 STAR-R
- 简历危险句兜底

### 业务 / 主管轮常见内容
- 45-60 秒自我介绍：偏业务理解和产品判断
- JD 要求逐条对应：简历哪段能证明
- 产品线 / 竞品 / 行业判断题
- 项目如何迁移到目标公司问题
- 面向业务负责人的反问

### 综合 / 终面常见内容
- 60 秒自我介绍：偏潜力、取舍、长期动机
- 复杂项目复盘：关键决策、冲突、取舍、失败教训
- "为什么是你"、"为什么我们"、"如果入职 30 天怎么做"
- 高层视角反问

### HR / recruiter 轮常见内容
- 动机、稳定性、到岗时间、薪资预期
- 背景短板解释：转行/非科班/空窗/经历跳跃
- offer / 实习周期 / 地点 / 签证等现实问题
- HR 面不能踩的雷

如果用户只准备某一轮，也至少产出 `00-总览.md` 和对应轮次文件。其他轮次只有在调研出现明确证据时，才列为"后续可能轮次/待确认"。

## 五、话术生成

### 自我介绍钩子（30 秒）
- 每个字都真，被任何追问都不崩
- 开头放最强钩子（王牌项目 + 一个亮眼真实数字）
- 针对真实轮次定制：简历深挖轮讲可信度，业务/主管轮讲业务判断，综合/终面讲潜力和取舍，HR/recruiter 轮讲动机和稳定性。

### 项目 STAR-R（每个王牌项目 60-90 秒）
**痛点（用户/场景/问题）→ 决策（2-3 个关键取舍 + 为什么）→ 执行（诚实说自己的角色）→ 结果（真实量化）→ 反思（产品认知）。**
决策写法区分高手和流水账：不说"我做了 X 功能"，说"我在 X 和 Y 之间选了 X，因为 Z"。

### Bullet 追问兜底表（防崩）
把第二三层问题提前备好，统一收敛到两条安全线：
| 追问类型 | 安全应答 |
|---------|---------|
| 技术实现细节 | "这部分我做的是产品/架构决策，原理大概是 X，但代码是 AI 辅助实现的，不是我从头敲的" |
| 数字怎么算 | "比起这个数字，我更想讲当时怎么做、为什么、业务上发生了什么改变" |
| 心虚经历被深挖 | 短答一句 + "这段时间短收获有限，我更想聊我后面做的王牌项目" |

> "AI 辅助"在今天是加分项，装工程师被追问三层一定崩。

### 软实力场景题（ToB/产品岗高频）
"需求 push 不动研发""研发说做不了""需求频繁变更"——核心：理解对方立场 + 用数据对齐 + 务实取舍，别答得太理想化。

### 反问（挑 2-3 个）
"这个岗位/组未来半年最重要的目标？" / "做得好的人最关键的特质？" / 针对具体产品："你们最近发的 X，背后的产品考量是？"
避免：一面问薪资/转正率、问能查到的基础信息。

## 六、面试准备文档结构

把以上组装成面试前 30 分钟能快速过的 `面试准备/` 文件夹。原则：**能背、能扫、认知负荷低**，多表格少长段落。

```
00-总览.md
01-简历bullet逐条深挖.md
02-{轮次名}.md
...
```

### Stage 3 完成标准
- [ ] `00-总览.md` 有轮次地图，且每个轮次都有依据、置信度和是否生成文件。
- [ ] `01-简历bullet逐条深挖.md` 覆盖最终简历的每一个 bullet，没有漏项。
- [ ] 每个 bullet 都有可直接说出口的 30 秒讲法、证据口径、追问和禁区。
- [ ] 只为 ✅ 已确认或 ⚠️ 高置信推断的轮次生成单独文件。
- [ ] 每个轮次文件都引用 bullet ID，并说明本轮为什么会问这些内容。
- [ ] ❓ 待确认轮次只在总览提醒，不生成完整话术，除非用户明确要求预备。
- [ ] 用户讲不清的 bullet 已标风险；简历未投则回改，已投则准备降级话术。

需要 PDF 用 pandoc 或 `pdf` skill 转；默认交付 Markdown。
