## 改了什么（一眼看懂）

**改之前**（弱版英文简历，全是辅助性描述）
```
Marketing Operations Intern | 2024 – Present
· Helped collect user feedback and keep track of tasks and messages.
· Did some operations work and wrote down notes for the team.
Event Sign-up Tool — AI Coding Project | 2026
· Built a simple web demo for event sign-up and check-in.
· Used AI coding tools to make pages, forms, and a database.
```

**改之后**（翻译成 AI 应用开发的语言：痛点 → 产品决策 → 可演示 Demo）
```
参赛定位：AI Coding 应用开发方向 | 擅长把日常业务问题转化为可执行的产品需求、
         数据字段、页面流程和可演示 Web Demo。

核心项目 · EventEase — 校园活动报名与签到 Web 工具
· 针对"报名表散落各处、签到靠人工点名、活动后统计累人"的痛点，做了一个集报名、
  签到、出勤统计于一体的轻量 Web 工具
· 设计核心数据模型：围绕 event / participant / signup status / check-in time
  设计表单、数据库表结构与接口，为出勤统计和导出沉淀结构化数据
· 用 Vibe Coding 从需求到 Demo：拆解页面流、表单字段、签到逻辑与边界场景

相关经历 · Marketing Operations Intern — 用户反馈与活动运营
· 把一线观察到的"信息分散、流程靠人工"等问题，转化为 EventEase 的字段、流程与低负担交互设计
```

## 关键手法

1. **加参赛/求职定位头**——开头一句话锁定方向
2. **把"做过的事"翻译成"产品能力"**——活动运营观察 → 产品需求洞察、数据建模、Demo
3. **新增一个真做出的项目当王牌**——用 Vibe Coding 真做一个可运行 Demo（EventEase）
4. **把面试准备拆细**——不是只写自我介绍，而是逐 bullet 准备证据、追问、表达状态和面后复盘题库

> 经历没有凭空捏造，全靠把已有的真实背景翻译到目标方向。
