# 表达状态与自我介绍 · 用户/内容运营实习

> 固定准备文件。这个候选人的关键不是把文科背景讲得很玄，而是稳定地证明自己真的做过内容、数据和复盘。

## 面试姿态

| 要做到 | 不要这样 |
|--------|----------|
| 说自己文科背景时语气平稳，马上接真实运营经历 | 一上来道歉式解释"我不是很对口" |
| 主动讲公众号里自己 own 的动作 | 只说"参与运营"等面试官追 |
| 数据记不清就说区间和后台口径 | 为了显得厉害硬报精确数字 |
| 文化机构实习短讲即可 | 把执行支持包装成完整增长项目 |

## 通用自我介绍（30 秒口语版）

> 您好，我是文科背景，但过去两年一直在做内容和活动运营。最完整的一段经历是学院公众号，我自己负责选题、标题、发布和数据复盘；也做过千人级活动传播，从报名入口到转发文案都有参与。我投用户/内容运营，是因为这类岗位需要理解用户、组织内容、看数据复盘，而这些是我已经真实做过的部分。

## 关键字卡

- 文科背景，不道歉
- 公众号：选题、标题、发布、复盘
- 活动传播：报名漏斗、转发文案
- 数据：打开率区间、后台口径
- 岗位匹配：用户理解 + 内容组织 + 复盘

## 项目逐字稿 · 校园公众号运营（90 秒）

| 模块 | 该怎么讲 |
|------|----------|
| 背景 | 学院公众号内容以前偏通知，打开率低，学生对很多活动信息感知弱。 |
| 我做了什么 | 我负责选题、标题、推送节奏和数据复盘，目标是让内容更像学生愿意点开的信息。 |
| 分步怎么做 | 先看历史推文表现，再把选题拆成通知类、经验类、活动类；标题上做更具体的利益点；发布后记录打开率和转发情况。 |
| 结果 | 打开率从个位数逐步到双位数区间；具体精确值以后台为准，不在现场硬报小数。 |
| 困难 | 学院内容天然不如娱乐内容有吸引力，所以不能只追热点，要把信息和学生利益挂上钩。 |
| 下次优化 | 会把用户反馈收集前置，比如用投票或评论区收集学生真正想看的主题。 |

## 不会就不会

| 被问到 | 安全回答 |
|--------|----------|
| 你能证明打开率提升完全由你造成吗？ | 我不能说完全由我单独造成，只能说我负责的标题、选题和发布节奏是主要调整项，数据变化和这些动作在时间线上对应。 |
| 你做过完整用户增长吗？ | 还没有做过商业产品里的完整增长闭环。我更准确的经验是校园内容和活动传播，可以迁移到用户洞察、内容组织和复盘。 |
| 文化机构实习有没有数据结果？ | 那段更偏内容执行和协作，数据结果不强。我会短讲，不把它包装成增长项目。 |

## 练习动作

- 录一遍 30 秒自我介绍，删掉所有道歉式开场。
- 公众号项目用 90 秒讲完，必须出现"目标、动作、数据、复盘"四个词。
- 随机抽两条运营 bullet，各讲 30 秒。
- 练习打开率口径：说区间、说后台来源，不硬报记不准的精确数。
